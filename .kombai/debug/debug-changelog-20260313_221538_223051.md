## Debug Changelog: Live Sessions Column Mismatch

**Session ID:** 20260313_221538_223051

## Issue Summary
The application was failing with error: `column "host_id" does not exist` when trying to query the `live_sessions` table. The root cause was a mismatch between the Drizzle ORM schema definition and the actual code using the table, combined with a missing database migration.

  - `getTutorLiveSessionInvite()` - changed `liveSessions.tutorId` to `liveSessions.hostId` (line 37) and `endedAt` to `actualEndedAt` (line 50)

## Root Cause Analysis
The schema file (`drizzle/schema/live-sessions.ts`) was defined correctly with `hostId` and `hostRole` fields, but:
1. Old code in multiple files still referenced `tutorId` (which doesn't exist in the new schema)
2. The actual PostgreSQL database table was never created - only the schema definition existed
3. This caused database query failures when the application tried to run
