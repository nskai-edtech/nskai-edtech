## Debug Session: 20260313_230214_798863

## Root Cause

The Clerk middleware (`proxy.ts`) explicitly allowed `SCHOOL_ADMIN` users to pass through `/tutor` routes via the `role !== "SCHOOL_ADMIN"` exception in the guard. However, all tutor server actions (e.g. `getTutorCourses`) check the database `role` field which is `"SCHOOL_ADMIN"` for those users, not `"TUTOR"`, causing them to throw `"Not a tutor"` → HTTP 500.

## Changes Made

### 1. Fixed `/tutor` route guard in middleware
- **File:** `proxy.ts`
- **Lines:** 143–149
- **Change:** Removed `&& role !== "SCHOOL_ADMIN"` from the `/tutor` route guard so `SCHOOL_ADMIN` users are correctly redirected to `/` instead of being allowed into the tutor dashboard.
- **Before:**
  ```typescript
  if (
    req.nextUrl.pathname.startsWith("/tutor") &&
    role !== "TUTOR" &&
    role !== "SCHOOL_ADMIN"
  ) {
  ```
- **After:**
  ```typescript
  if (
    req.nextUrl.pathname.startsWith("/tutor") &&
    role !== "TUTOR"
  ) {
  ```
- **Revert:** Add back `&& role !== "SCHOOL_ADMIN"` on its own line between `role !== "TUTOR"` and the closing `)`
