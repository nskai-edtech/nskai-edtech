# Debug Changelog — 20260313_232358_569490

## Root Cause

`schedule-session-modal.tsx` is a `"use client"` component that directly imported
`notifyLearnersOfScheduledSession` from `lib/live-sessions/notifications.ts`.
That file imports `db` from `lib/db.ts`, which caused **`lib/db.ts` to be bundled
into the client-side JavaScript**.

On the client, `process.env.DATABASE_URL` is `undefined` (it is not a `NEXT_PUBLIC_`
variable), so `neon(process.env.DATABASE_URL!)` received `undefined` and threw:

> No database connection string was provided to `neon()`. Perhaps an environment variable has not been set?
