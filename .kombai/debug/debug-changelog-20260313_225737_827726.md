## Root Cause

The TUTOR modal in `app/onboarding/page.tsx` was a placeholder stub. Its submit button called `onClick={() => alert("Submit Code")}` and **never called `completeOnboarding()`**, so no role was ever saved to the user's Clerk metadata.

Without a role, the middleware (`proxy.ts`) kept the user on `/onboarding`. If the user — unable to proceed — clicked "Register My School" instead, the working `OnboardingWizard` would assign them `role: SCHOOL_ADMIN`. On next sign-in, the middleware correctly redirected `SCHOOL_ADMIN` → `/school-dashboard`, which was the reported bug.

The routing logic in `proxy.ts` was **correct**. The bug was entirely in the broken tutor form.
